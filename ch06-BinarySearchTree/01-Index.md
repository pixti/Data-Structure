## 1. 색인 (Index)과 검색 트리
데이터의 양이 폭발적으로 증가함에 따라, 원하는 정보를 빠르게 찾아내기 위한 '색인'과 이를 구현하는 '검색 트리'의 중요성이 커지고 있습니다.  

### 1-1. 데이터의 홍수와 색인의 필요성
현대 사회는 매일 엄청난 양의 데이터가 생성되는 '데이터의 홍수' 시대입니다.  

* **데이터 생성량**: 매일 약 250경($2.5 \times 10^{18}$) bytes의 데이터가 발생합니다.  
* **규모 체감**: 이는 1TB(Tera byte) 하드디스크 250만 개 분량에 해당합니다.  
* **해결책 (색인)**: 방대한 데이터 속에서 특정 항목을 효율적으로 찾기 위해 책의 뒷부분 '찾아보기'와 같은 **색인(Index)** 개념이 웹 사이트, 파일, 레코드 관리에 필수적으로 사용됩니다.  

### 1-2. 레코드, 키(Key), 그리고 색인의 목적  
데이터베이스와 자료구조에서 정보를 다루는 기본 단위는 레코드입니다.  

#### ① 주요 용어 정리  
* **레코드(Record)**: 개체에 대한 모든 정보를 포함하는 단위입니다.  
    * 예: '사람' 레코드 (주민번호, 이름, 주소, 연락처, 학력 등 포함)  
* **필드(Field)**: 레코드를 구성하는 각각의 세부 항목입니다.  
* **키(Key)**: 레코드를 유일하게 식별할 수 있는 필드입니다. (예: 주민등록번호, 사번)  

> **색인의 핵심 목적**  
> 색인의 궁극적인 목적은 특정 키를 가진 **개체의 레코드를 신속하게 검색**하는 것입니다.  

### 1-3. 색인 추상 데이터 타입 (Index ADT)  
색인이 수행해야 하는 작업들을 추상화하면 다음과 같은 연산들로 정의할 수 있습니다.  

* **삽입**: 색인에 키 $x$를 삽입한다.  
* **검색**: 색인에서 키 $x$를 검색한다.  
* **삭제**: 색인에서 키 $x$를 삭제한다.  
* **상태 확인**: 색인이 비어 있는지 확인하거나, 색인을 깨끗이 비운다.  

### 1-4. 자료구조별 효율성 비교  
배열이나 연결 리스트도 색인으로 사용할 수 있지만, 데이터 규모가 커지면 효율성이 급격히 떨어집니다.   
 
| 자료구조 | 방식 | 랜덤 삽입 | 랜덤 삭제 | 랜덤 검색 |  
| :--- | :--- | :--- | :--- | :--- |  
| **배열** | 선착순 | $O(1)$ | $O(N)$ | $O(N)$ |   
| | 정렬 | $O(N)$ | $O(N)$ | $O(\log N)$ |  
| **연결 리스트** | 선착순 | $O(N)$ | $O(N)^*$ | $O(N)$ |   
| | 역순 | $O(1)$ | $O(N)^*$ | $O(N)$ |   
| | 정렬 | $O(N)^*$ | $O(N)^*$ | $O(N)$ |  

$^*$ *검색 후 삽입/삭제 위치를 이미 알고 있는 경우 O(1) 가능*  

### 1-5. 검색 트리의 분류와 활용  
색인을 효율적으로 구현하기 위해 다양한 '검색 트리'가 사용됩니다. 이는 크게 저장 위치와 자식의 수에 따라 분류됩니다.  

#### ① 저장 위치에 따른 분류
* **내장 색인 (Main Memory Index)**: 데이터 규모가 메모리 용량보다 작을 때 사용합니다.   
    * 주요 구조: 이진 검색 트리, AVL 트리, 레드-블랙 트리   
* **외장 색인 (External Index)**: 데이터가 너무 커서 디스크(Disk)에 저장해야 할 때 사용합니다.    
    * 주요 구조: B-트리, 2-3-4 트리

* **이진 검색 트리 (Binary Search Tree)**: 각 노드가 최대 2개의 자식을 가집니다. (AVL, Red-Black 등)  
* **다진 검색 트리 (Multi-way Search Tree)**: 각 노드가 3개 이상의 자식을 가질 수 있어 트리의 높이를 낮출 수 있습니다. (B-트리, 2-3-4 트리)   

> **균형 검색 트리 (Balanced Search Tree)**  
> 트리 기반 색인에서 가장 중요한 것은 **균형**입니다. AVL 트리, 레드-블랙 트리, B-트리는 삽입/삭제 시 스스로 균형을 맞춰 최악의 경우에도 $O(\log N)$ 의 성능을 보장합니다.
