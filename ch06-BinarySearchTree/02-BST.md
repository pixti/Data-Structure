## 2. 이진 탐색 트리 (Binary Search Tree, BST)
이진 탐색 트리는 **이진 탐색(Binary Search)** 의 효율적인 탐색 로직을 트리 구조에 접목하여,   
데이터의 삽입과 삭제가 빈번한 상황에서도 빠른 검색을 유지하도록 설계된 자료구조입니다.  

### 2-1. 정의 및 성질 (Formal Definition)  
이진 탐색 트리는 이진 트리(Binary Tree)의 일종으로, 모든 노드 $n$에 대해 다음의 **수학적 조건** 을 만족해야 합니다.

* **좌측 서브트리 조건**: 노드 $n$의 왼쪽 서브트리에 있는 모든 노드의 키값은 노드 $n$의 키값보다 **작아야 함** ( $Key(L) < Key(n)$ )  
* **우측 서브트리 조건**: 노드 $n$의 오른쪽 서브트리에 있는 모든 노드의 키값은 노드 $n$의 키값보다 **커야 함** ( $Key(R) > Key(n)$ )  
* **재귀적 성질**: 모든 서브트리 역시 그 자체로 이진 탐색 트리여야 함  

### 2-2. 주요 특징과 탐색 기법  
이진 탐색 트리는 데이터가 정렬된 상태로 유지되는 특성 덕분에 다음과 같은 유용한 성질을 가집니다.  

#### ① 중위 순회(Inorder Traversal)와 정렬  
이진 탐색 트리를 **중위 순회(Inorder Traversal, L->V->R)** 하면 저장된 키값들이 **오름차순으로 정렬**되어 출력됩니다.  
* 이는 별도의 정렬 알고리즘 없이도 정렬된 데이터를 얻을 수 있음을 의미합니다.  

#### ② 최소값과 최대값 찾기  
* **최소값(Minimum)**: 루트에서 시작하여 `null`을 만날 때까지 계속해서 **왼쪽 자식**을 따라 내려가면 만나는 노드입니다.  
* **최대값(Maximum)**: 루트에서 시작하여 `null`을 만날 때까지 계속해서 **오른쪽 자식**을 따라 내려가면 만나는 노드입니다.  

### 2-3. 트리 구조에 따른 효율성 분석  
이진 탐색 트리의 성능은 트리의 **높이($H$)** 에 직접적으로 의존합니다. 동일한 데이터 세트라도 삽입되는 순서에 따라 트리의 모양과 성능이 극명하게 갈립니다.  

| 트리 모양 | 특징 | 탐색 시간 복잡도 |  
| :--- | :--- | :--- |  
| **최선의 모양 (Balanced)** | 좌우 균형이 완벽하게 잡힌 상태 | **$O(\log N)$** |  
| **일반적인 모양** | 어느 정도 균형이 잡힌 상태 | $O(H)$ |  
| **최악의 모양 (Skewed)** | 한쪽으로 치우친 선형 구조 (연결 리스트와 동일) | **$O(N)$** |  

#### 시간 복잡도와 높이의 관계  
* 트리가 균형을 이룰 때, 노드 수가 $N$이면 높이 $H \approx \log_2 N$이 됩니다.  
* 하지만 정렬된 데이터를 그대로 삽입할 경우 트리가 한쪽으로 길게 늘어지는 **경사 이진 트리(Skewed Tree)** 가 형성되어, 검색 효율이 배열의 순차 탐색 수준으로 떨어지게 됩니다.  

### 2-4. 이진 탐색(Array) vs 이진 탐색 트리(Tree)  
둘 다 이진 탐색 원리를 사용하지만, 데이터 유지 관리 측면에서 차이가 있습니다.  

* **이진 탐색 (정렬 배열)**: 고정된 크기 내에서 매우 빠르지만($O(\log N)$), 새로운 데이터를 삽입하거나 삭제할 때 배열의 요소를 밀거나 당기는 $O(N)$의 비용이 발생합니다.    
* **이진 탐색 트리**: 트리가 균형을 유지한다는 전제하에 **탐색, 삽입, 삭제 모두 $O(\log N)$**에 수행할 수 있어 동적인 데이터 관리에 훨씬 유리합니다.
